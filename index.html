JENKINS CI


Jenkins CI is a very popular continuous integration and continuous delivery tool. Jenkins offers support for a huge range of languages and source code systems with plugins for integrations to automate all types of tasks and and huge range of plugin extensions. It allows developers to create and visualise continuous delivery pipelines, automate testing and build processes.


Click on the icons above to learn how Jenkins works

JENKINS CI SETUP


Download and install Jenkins from https://jenkins.io/download. Open the terminal and navigate to the folder with the jenkins.war file in it.
Paste the following to start jenkins on localhost:8080 or an alternative port:
java -jar jenkins.war --ajp13Port=-1 --httpPort=8080

Open browser at the specified port eg. localhost:8080. You will be prompted to add your authentication code given in the terminal, then go ahead and create an account. After logging in, you will initially be taken to the home page with the original UI.

JENKINS PLUGINS

One of the main features of Jenkins is it's large range of plugins and ability to extend functionality.


Jenkins UI

Pipelines can be created with the original UI or new version known as Blue Ocean. This provides an improved user experience with pipeline visualisation, guiding users through the process to quickly and easily create pipelines.


The steps below will run through managing and installing plugins with the example of Blue Ocean.

First select 'Manage Jenkins' in the navigation and from the next page select 'Manage Plugins'.

On the following page you will see a number of tabs including 'Updates, Available, Installed and Advanced', select 'Available'. Scrolling through you will see a huge range of available plugins. Use the filter bar to search for 'Blue Ocean' or 'Ocean' in the search bar.

Select the main 'Blue Ocean' plugin from the dropdown and select 'Install without restart' or 'Download now and install after restart'.

Once Succesfully installed, Blue Ocean will appear in the navigation bar. For details on how to use Blue Ocean, select the pipeline icon above.

JENKINSFILE


The Jenkinsfile contains information about the pipeline process including build, test, deployment, agents, workspace, stages and steps.

When you make changes to your timeline, the Jenkinsfile is updated with changes commited to your version control system.

JENKINS PIPELINE

The Jenkins Pipeline allows users to create and visualise continuous delivery pipelines detailing and automating the build, testing and deployment processes.


From the Blue Ocean home page you will see a list of all your pipelines. To create a new one, click the 'New Pipeline' button in the top right.

Next select where your code is stored eg, GitHub. You will be prompted to enter your authorisation token which you will need to retrieve from the version control system. Once entered and connected, select the organisation, repository and click 'Create Pipeline.'

Choose the organisation the repository belongs to and repository name. You will then be asked if you want to create a new Pipeline or create one from an existing Jenkinsfile. Upon clicking create new pipeline, you will be taken to a new page where you are able to visualise the pipeline and add stages and steps.

To add a new stage to the pipeline simply click the + symbol on the left and fill in the details in the box on the right. You will first be asked to name your stage eg. Build, Test, Deploy.

You can then add Pipeline settings such as the Agent eg. Docker. You will then be required to add steps to the stage by clicking the 'Add Step' button. You will receive and error if no steps are added successfully. Select from the dropdown box the method of you step eg.'Print message' and enter appropriate details such as the message.

When you have completed your pipeline, click the 'Save' button in the top right.

If you wish to view/edit the script, pressing 'Command + s' on a mac or 'ctrl + s' on windows and a pop up will appear.

The save button will commit a Jenkinsfile, first a modal will appear which will ask for a description of changes and choice of commit to master or new branch. You can then save and run your pipeline.

Next you will be taken to a page listing all your pipelines with details about the status, number of runs, message, repository branch, duration and when completed.

Click on the Pipeline to view further details including Pipeline Stages, ouput and time taken, Changes/commits, Tests, Artifacts with pipeline log.
